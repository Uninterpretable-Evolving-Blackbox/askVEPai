## User Query
annotate a large cohort VCF with population allele frequencies for filtering

## Recommendation
### 1. Detected Use Case
**Category:** `population_genetics`
**Reason:** The user's explicit goal is to annotate a large cohort with "population allele frequencies" specifically for the purpose of "filtering." This matches the core objective of population genetics analysis, which relies on identifying common vs. rare variants using databases like gnomAD and 1000 Genomes.

### 2. Recommended Options
✓ **core_type** [source: core_type, priority=critical] confidence: high
  Reason: Standardizes the annotation against the Ensembl/GENCODE reference transcripts.

✓ **symbol** [source: symbol, priority=recommended] confidence: high
  Reason: Provides human-readable gene symbols (e.g., BRCA1), making it easier to interpret variants after filtering.

✓ **check_existing** [source: check_existing, priority=critical] confidence: high
  Reason: Essential prerequisite for any frequency-based annotation; enables VEP to find and report co-located known variants and rsIDs.

✓ **af_gnomade** [source: af_gnomade, priority=critical] confidence: high
  Reason: Provides gnomAD exome allele frequencies, which are critical for identifying rare variants in coding regions.

✓ **af_gnomadg** [source: af_gnomadg, priority=critical] confidence: high
  Reason: Provides gnomAD genome allele frequencies, necessary for assessing the frequency of variants in non-coding/genomic regions.

✓ **af** [source: af, priority=critical] confidence: high
  Reason: Includes 1000 Genomes Phase 3 global allele frequencies to identify common variants globally.

✓ **af_1kg** [source: af_1kg, priority=critical] confidence: high
  Reason: Provides super-population-specific frequencies (AFR, AMR, EAS, EUR) from 1000 Genomes, allowing for filtering of population-specific expansion/drift.

✓ **frequency** [source: frequency, priority=recommended] confidence: high
  Reason: Enables the actual pre-filtering logic within VEP based on the allele frequencies retrieved by the other flags.

✗ **sift** [source: sift] confidence: high
  Reason: Pathogenicity prediction is not requested; disabling this reduces computational overhead for a large cohort.

✗ **polyphen** [source: polyphen] confidence: high
  Reason: Function prediction adds significant runtime and is unnecessary if the goal is purely frequency-based filtering.

✗ **cadd** [source: cadd] confidence: high
  Reason: CADD is computationally intensive for large datasets and out of scope for a frequency-focused task.

✗ **most_severe** [source: most_severe] confidence: medium
  Reason: While it simplifies output, it may hide transcript-specific frequency information that might be useful during secondary analysis; keeping all transcripts allows for more granular filtering.

### 3. Generated VEP Command
```bash
vep --input_file cohort.vcf --output_file annotated_cohort.txt --cache --check_existing --symbol --af_gnomade --af_gnomadg --af --af_1kg --frequency
```
*Note: On the Ensembl Web Interface, ensure "Check for existing variants" is checked and select all gnomAD/1000 Genomes frequency options in the "Frequency" section. For large cohorts, it is highly recommended to run this via the Command Line Interface (CLI) using a local cache to avoid web interface timeouts.*
