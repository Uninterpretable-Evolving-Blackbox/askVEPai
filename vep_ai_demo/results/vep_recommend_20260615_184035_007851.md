## User Query
rare disease patient exome, suspected Mendelian disorder, prioritise pathogenic variants

## Recommendation
### 1. Detected Use Case
**Category:** `rare_disease_germline`
**Reason:** The user specifies a "rare disease patient exome" and a search for "Mendelian disorder"-related pathogenicity. This requires high-sensitivity annotations, clinical significance lookups (ClinVar), population frequency filters (gnomAD) to exclude common variants, and advanced protein-effect predictors.

### 2. Recommended Options
✓ symbol [source: symbol, priority=recommended] confidence: high
  Reason: Essential for human-readable results by mapping Ensembl IDs to HGNC gene symbols.

✓ hgvs [source: hgvs, priority=critical] confidence: high
  Reason: Provides standardized nomenclature required for clinical reporting and variant communication.

✓ mane [source: mane, priority=critical] confidence: high
  Reason: Identifies MANE Select transcripts, ensuring the analysis aligns with NCBI/Ensembl standards for clinical relevance.

✓ canonical [source: canonical, priority=recommended] confidence: high
  Reason: Helps prioritize the most biologically relevant transcript for each gene.

✓ check_existing [source: check_existing, priority=critical] confidence: high
  Reason: Necessary to link input variants to known dbSNP IDs and ClinVar entries.

✓ clinvar [source: clinvar, priority=critical] (via check_existing) confidence: high
  Reason: Directly provides clinical significance (Pathogenic/Benign) which is the primary goal of the analysis.

✓ af_gnomade [source: af_gnomade, priority=critical] confidence: high
  Reason: Allows filtering out common variants that are unlikely to cause rare Mendelian diseases.

✓ af_gnomadg [source: af_gnomadg, priority=critical] confidence: high
  Reason: Provides genome-wide frequency data to complement exome-specific frequencies.

✓ sift [source: sift, priority=critical] confidence: high
  Reason: Standard tool for predicting the impact of amino acid substitutions on protein function.

✓ polyphen [source: polyphen, priority=critical] confidence: high
  Reason: Complementary to SIFT; provides another layer of functional prediction for missense variants.

✓ dbnsfp [source: dbnsfp, priority=recommended] confidence: high
  Reason: Accesses an aggregate resource of many pathogenicity scores (CADD, etc.) in one place.

✓ revel [source: revel, priority=recommended] confidence: high
  Reason: An ensemble method that provides a highly accurate single score for missense pathogenicity.

✓ alphamissense [source: alphamissense, priority=recommended] confidence: high
  Reason: Uses DeepMind’s latest models to provide state-of-the-art predictions for missense effects.

✓ cadd [source: cadd, priority=recommended] confidence: high
  Reason: Provides a unified score of deleteriousness across different types of variants.

✓ loeuf [source: loeuF, priority=recommended] confidence: medium
  Reason: Helps identify genes that are intolerant to loss-of-function, aiding in the prioritization of rare LoF variants.

✓ coding_only [source: coding_only, priority=optional] confidence: high
  Reason: Since this is an exome analysis, restricting output to coding regions reduces noise and computational load.

✓ biotype [source: biotype, priority=recommended] confidence: medium
  Reason: Allows the user to filter for protein-coding transcripts specifically.

✓ pick [source: pick, priority=recommended] confidence: high
  Reason: Reduces redundant output by selecting a single representative transcript/consequence per variant, making it easier to review.

✗ most_severe [source: most_severe] confidence: high
  Reason: Disabled because using `--most_severe` conflicts with the ability to see detailed individual predictions like SIFT and PolyPhen for all transcripts.

✗ summary [source: summary] confidence: high
  Reason: Disabled because a summary view is too reductive; clinical interpretation requires viewing the full details of the predicted impact.

### 3. Generated VEP Command
```bash
vep --input_file patient_exome.vcf --output_file analysis_results.txt --cache \
--symbol --hgvs --check_existing --clinvar --af_gnomade --af_gnomadg \
--sift --polyphen --dbnsfp --revel --AlphaMissense --CADD --plugin LOEUF \
--biotype --coding_only --mane --canonical --pick
```

**Note for Web Interface Users:**
To replicate this setup in the Ensembl VEP web tool:
1. Set **Restrict results** to `pick`.
2. Enable **HGVS**, **Symbol**, and **Protein** (if available).
3. Under **Plugin**, select/enable **dbNSFP**, **CADD**, **REVEL**, and **AlphaMissense**.
4. Ensure **Transcript flags** like **MANE Select** and **Canonical** are checked.
5. In the **Frequency** section, ensure **gnomAD** data is included.
