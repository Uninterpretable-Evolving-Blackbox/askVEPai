# VEP Recommendation

## User Query
germline exome variants from a rare disease patient, find pathogenic coding variants

## Recommendation
### 1. Detected Use Case
**Category:** `rare_disease_germline`
**Reason:** The user is analyzing germline exome data from a patient with a suspected Mendelian disorder, specifically aiming to identify pathogenic coding variants for clinical interpretation.

### 2. Recommended Options
✓ **core_type** [source: core_type, priority=critical] confidence: high
  Reason: Essential for defining the underlying gene/transcript annotation set (Ensembl default) used for consequence calculation.

✓ **symbol** [source: symbol, priority=recommended] confidence: high
  Reason: Provides human-readable gene symbols (e.g., BRCA1), which are essential for clinical identification of genes.

✓ **hgvs** [source: hgvs, priority=critical] confidence: high
  Reason: Adds HGVS nomenclature, standard for communicating variants to clinicians and laboratories.

✓ **mane** [source: mane, priority=critical] confidence: high
  Reason: Flags MANE Select/Plus transcripts, ensuring the user focuses on the high-confidence transcripts used across clinical databases.

✓ **canonical** [source: canonical, priority=recommended] confidence: high
  Reason: Helps prioritize the most biologically relevant transcript for each gene.

✓ **protein** [source: protein, priority=optional] confidence: high
  Reason: Links variants to their corresponding protein IDs (ENSP), facilitating downstream functional analysis.

✓ **check_existing** [source: check_existing, priority=critical] confidence: high
  Reason: Necessary to look up co-located known variants, which is the foundation for ClinVar and frequency reporting.

✓ **clinvar** [source: clinvar, priority=critical] confidence: high
  Reason: Provides clinical significance (e.g., Pathogenic/Benign) from ClinVar, which is crucial for rare disease interpretation.

✓ **af_gnomade** [source: af_gnomade, priority=critical] confidence: high
  Reason: Allows filtering of common variants using gnomAD exome frequencies to isolate rare disease-causing mutations.

✓ **af_gnomadg** [source: af_gnomadg, priority=critical] confidence: high
  Reason: Provides genome-wide allele frequencies to complement exome data and identify potential low-frequency variants.

✓ **sift** [source: sift, priority=critical] confidence: high
  Reason: Provides a widely used prediction of whether an amino acid substitution affects protein function.

✓ **polyphen** [source: polyphen, priority=critical] confidence: high
  Reason: Essential tool for predicting the impact of missense mutations on protein structure and function.

✓ **cadd** [source: cadd, priority=recommended] confidence: high
  Reason: Provides an integrated score of deleteriousness, helping to rank variants by potential impact.

✓ **coding_only** [source: coding_only, priority=optional] confidence: medium
  Reason: The user specifically requested "coding variants"; this flag filters out non-coding consequences to streamline the analysis.

✓ **spliceai** [source: spliceai, priority=recommended] confidence: medium
  Reason: Even in exome data, predicting the impact on splice sites (e.g., cryptic/exon-junction) is vital for identifying pathogenic variants.

✗ **most_severe** [source: most_severe] confidence: high
  Reason: Must be disabled because clinical review requires seeing all transcript-level consequences to assess total protein impact and avoid missing details.

✗ **summary** [source: summary] confidence: high
  Reason: Must be disabled because a collapsed summary prevents the detailed annotation needed for pathogenicity assessment.

### 3. Generated VEP Command
```bash
vep --input_file variants.vcf --output_file annotated_variants.txt --cache --format vcf --everything \
--coding_only \
--mane \
--canonical \
--symbol \
--hgvs \
--protein \
--check_existing \
--clinvar \
--af_gnomade \
--af_gnomadg \
--sift B \
--polyphen P \
--plugin CADD,file=/path/to/CADD_scores.tab.gz \
--plugin SpliceAI,file=/path/to/spliceai_predictions.tar.gz
```
*Note: `--everything` is a shorthand for many standard flags, but manual inclusion of specific plugins like CADD and SpliceAI with their file paths (as shown above) is required for local execution.*
