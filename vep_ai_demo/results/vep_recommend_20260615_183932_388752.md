## Recommendation
### 1. Detected Use Case
**Category:** `non_human`
**Reason:** The analysis involves the mouse genome (GRCm39), which precludes the use of human-specific clinical and population databases (e.g., ClinVar, gnomAD).

### 2. Recommended Options
✓ core_type [source: core_type, priority=essential] confidence: high
  Reason: Essential to define the underlying transcript/gene annotation set for GRCm39.

✓ symbol [source: symbol, priority=recommended] confidence: high
  Reason: Adds HGNC-like symbols for mouse genes (e.g., *Tp53*), making results human-readable and much easier to interpret in a knockout context.

✓ biotype [source: biotype, priority=recommended] confidence: high
  Reason: Identifies whether the target is protein-coding, lincRNA, etc., which is critical for assessing the impact of CRISPR-mediated deletions/insertions.

✓ hgvs [source: hgvs, priority=recommended] confidence: high
  Reason: Provides standardized nomenclature (e.g., c.123C>T) to precisely track the engineered mutation across different tools.

✓ check_existing [source: check_existing, priority=recommended] confidence: high
  Reason: Allows identification of known mouse polymorphisms or existing dbSNP entries co-located with your CRISPR indels.

✓ domains [source: domains, priority=recommended] confidence: high
  Reason: Adds protein domain information (Pfam/PROSITE), which is vital for assessing if a knockout disrupts a functional protein unit.

✓ ccds [source: ccds, priority=optional] confidence: medium
  Reason: Adds CCDS identifiers where available; helpful for cross-referencing mouse transcripts between Ensembl and NCBI.

✓ nmd [source: nmd, priority=recommended] confidence: high
  Reason: Predicts if a mutation will trigger nonsense-mediated decay, a critical mechanism to consider in knockout experiments involving premature stop codons.

✓ numbers [source: numbers, priority=optional] confidence: medium
  Reason: Provides exon/intron numbering (e.g., 2/10) to help localize the CRISPR edit within the gene structure.

✗ polyphen [source: polyphen] confidence: high
  Reason: PolyPhen is a human-only tool and cannot be used on mouse data.

✗ cadd [source: cadd] confidence: high
  Reason: CADD integrates multiple annotations specific to human genomic features and is unavailable for mouse.

✗ clinvar [source: clinvar] confidence: high
  Reason: ClinVar provides clinical significance for human variants; it does not contain mouse-specific disease data.

✗ af_gnomade [source: af_gnomade] confidence: high
  Reason: gnomAD is a human-only population database.

✗ af_gnomadg [source: af_gnomadg] confidence: high
  Reason: gnomAD genome populations are specific to humans.

✗ revel [source: revel] confidence: high
  Reason: REVEL is an ensemble method trained on human disease-causing variants and is not applicable here.

### 3. Generated VEP Command
```

```

Ensure you select **mm39 (GRCm39)** as the species in the dropdown. To replicate the command above, check the boxes for **Gene Symbol**, **Transcript Biotype**, **HGVS**, **Check existing variants**, **Protein domains**, **CCDS**, and **NMD**. Avoid selecting "ClinVar" or "gnomAD".
